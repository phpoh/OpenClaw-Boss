# 🦞 OpenClaw 增长与扩展 — 增长策略与项目组合管理

分析项目增长态势，制定扩展策略，管理多项目组合。

## 参数

- `$ARGUMENTS` — 必填，项目/公司运营数据描述。
  示例：`SaaS项目A: 月收入$32000, 增长+15%MoM; API项目B: 月收入$12000, 增长+3%MoM`
  或描述具体情况，如 `我的项目连续2个月增长停滞，月收入$5000，用户增长<5%`

## 执行步骤

### 第一阶段：项目状态评估

1. 解析输入的项目数据
2. 根据决策矩阵判定每个项目状态：

| 状态 | 条件 | 行动 |
|------|------|------|
| 📈 GROW | 月增长 > 10% 且 LTV/CAC > 3 且跑道 > 6月 | 加大投入 |
| ⏸️ HOLD | 月增长 0-10% 且 LTV/CAC > 1.5 且跑道 > 6月 | 保持优化 |
| 🔍 REVIEW | 连续2月负增长 或 LTV/CAC < 1.5 或跑道 < 6月 | 诊断转型 |
| 🚪 EXIT | 连续3月 burn > revenue×2 且无盈利路径 | 退出清算 |

3. 输出项目状态仪表盘（表格）

### 第二阶段：市场环境分析

1. 使用 WebSearch 分析当前市场环境：
   - "{行业} market outlook 2026"
   - "{行业} growth forecast"
   - "{行业} competitive landscape"

2. 识别外部机会和威胁

### 第三阶段：增长策略推荐

根据项目状态推荐策略：

**📈 GROW — 加速增长：**
- 新客户群体定位建议
- A/B 测试定价方案（3个价格点）
- 营销渠道扩展建议
- 产品线扩展评估

**⏸️ HOLD — 效率优化：**
- 转化率优化建议
- 运营自动化机会
- 成本削减方案
- 客户成功流程改进

**🔍 REVIEW — 诊断转型：**
- 增长瓶颈诊断
- 转型方向建议
- Pivot 方案制定
- 时间窗口评估

**🚪 EXIT — 优雅退出：**
- 退出时间线（2-4周计划）
- 资产处置方案
- 人员安置建议
- 经验教训提取

### 第四阶段：资源分配优化

如果是多项目组合，制定资源分配方案：

```
分配规则：
  GROW 项目 → 60% 新增资源
  HOLD 项目 → 25% 新增资源
  REVIEW 项目 → 15% 新增资源
  EXIT 项目 → 0% 新增资源，逐步释放
```

输出资源分配表：

| 项目 | 状态 | 收入/月 | 增长率 | 资源占比 | 建议行动 |
|------|------|---------|--------|---------|---------|

### 第五阶段：扩展机会识别

1. 使用 WebSearch 识别扩展机会：
   - "{当前领域} adjacent markets"
   - "{当前领域} expansion opportunities"
   - "{现有能力} leverage new products"

2. 评估每个扩展机会：
   - 可复用的技术/团队/客户资源
   - 新市场进入门槛
   - 预期 ROI

### 第六阶段：战略路线图

生成 90 天战略路线图：

```
第1-30天：[具体行动]
第31-60天：[具体行动]
第61-90天：[具体行动]
```

标注关键里程碑和需要人类审批的决策点。

### 输出格式

请输出完整的增长与扩展策略报告，包含以上六个阶段。使用表格、列表和可视化文字呈现。如果是多项目组合，输出组合仪表盘。最后给出一个综合建议（1-3 句话）。
