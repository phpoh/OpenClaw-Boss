# 🦞 OpenClaw 产品构建 — 产品开发全流程规划

为已验证的商业机会制定产品开发计划，从规格定义到部署上线的完整流程。

## 参数

- `$ARGUMENTS` — 必填，产品需求描述，如 `AI错题本SaaS，支持拍照识别+自动归类+智能推荐练习`。

## 执行步骤

### 第一阶段：规格定义

1. 分析产品需求，转化为用户故事：
   - 使用 "作为{角色}，我想{功能}，以便{价值}" 格式
   - 定义验收标准（Acceptance Criteria）
   - 按 MoSCoW 方法排优先级（Must / Should / Could / Won't）

2. 输出产品需求文档（PRD）大纲

### 第二阶段：技术架构设计

1. 使用 WebSearch 调研技术方案：
   - "{产品描述} 技术架构 技术选型"
   - "{产品描述} 开源方案"
   - "{产品描述} 最佳实践"

2. 根据项目类型推荐技术栈：

| 项目类型 | 前端 | 后端 | 数据库 | 部署 |
|---------|------|------|--------|------|
| Web SaaS | Vue3 + Element Plus / React + Ant Design | Spring Boot / FastAPI / Node.js | MySQL + Redis | 阿里云 / 腾讯云 |
| 移动优先 | uni-app / Flutter | Spring Boot / Go | MySQL | 阿里云 + CDN |
| 小程序 | 微信小程序 / uni-app | Spring Boot / Node.js | MySQL + Redis | 腾讯云 |
| API 服务 | - | Spring Boot / Go | MySQL + Redis | 阿里云函数计算 |
| 数据平台 | Vue3 + ECharts | Python | ClickHouse + MySQL | 阿里云 + DataWorks |

3. 输出架构概览（文字描述组件关系）

### 第三阶段：任务拆解与排期

1. 将用户故事拆解为可执行任务（每个 ≤ 4 小时）
2. 排定 Sprint 计划（7 天一个 Sprint）
3. 标注哪些任务适合 AI 编码，哪些需要人类

输出任务看板：

| 任务 | 优先级 | 预估工时 | 执行者(AI/人类) | Sprint | 依赖 |
|------|--------|---------|---------------|--------|------|

### 第四阶段：质量保障计划

定义测试策略：

```
测试金字塔：
    ╱  E2E 测试  ╲       ← 核心用户路径
   ╱   集成测试    ╲      ← API 和模块间
  ╱    单元测试      ╲     ← 核心逻辑

标准：
  - 单元测试覆盖率 > 80%
  - API 集成测试通过率 100%
  - P95 延迟 < 500ms
  - 无高危安全漏洞
```

### 第五阶段：部署策略

制定部署计划：

1. **部署方式**：推荐金丝雀 → 蓝绿 → 全量
2. **基础设施**：推荐云服务（阿里云/腾讯云/华为云）
3. **域名与备案**：ICP备案流程、域名解析配置
4. **监控**：阿里云ARMS / 腾讯云监控，关键指标和告警规则
5. **回滚方案**：回滚触发条件和操作步骤

### 第六阶段：衡量指标

定义上线后的核心指标：

| 指标 | 目标值 | 衡量方式 |
|------|--------|---------|
| 用户注册量 | 第1周 > 100 | 神策/友盟埋点 |
| 核心功能使用率 | > 50% | 行为分析 |
| NPS | > 20 | 用户调研 |
| 日活/周活 | WAU/注册 > 30% | 神策/友盟埋点 |

### 输出格式

请输出完整的产品开发计划，包含以上六个阶段的结果。最后生成一个里程碑时间线（按天/周），标注关键节点和需要人类审批的检查点。
