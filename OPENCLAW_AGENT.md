## 技能 1：👀 market-sense（市场感知）

**触发词：** 扫描市场 / 市场感知 / market-sense / 发现机会

**执行步骤：**

1. **数据采集** — 使用 WebSearch 搜索用户指定领域（默认 technology, saas, consumer, healthcare）的最新趋势、用户痛点、未满足需求
   - 搜索关键词："{领域} emerging needs 2026"、"{领域} unsolved problems"、"{领域} startup opportunities"
   - 搜索社区：ProductHunt、Reddit、HackerNews 等平台上的热门讨论

2. **噪声过滤** — 去除广告、重复、低质量内容，只保留与目标领域强相关且含数据支撑的洞察

3. **模式识别** — 将发现归类为 5 种模式：
   - 📈 上升趋势（持续增长）、🔥 突发热点（短期爆发）、😤 痛点聚集（反复出现）、🕳️ 市场空白（有需求无方案）、⚡ 技术拐点（新技术使不可能变可能）

4. **机会评分** — 按公式打分：
   ```
   score = market_size(0.25) + growth_rate(0.20) + competition_gap(0.20) + feasibility(0.15) + alignment(0.10) + timing(0.10)
   ```
   - 0.80-1.00 🔥 紧急 → 立即验证
   - 0.60-0.80 🟢 高优先 → 建议执行 demand-validate
   - 0.30-0.60 🟡 中等 → 列入候选
   - 0.00-0.30 🔴 低 → 存档观察

**输出：** 机会列表（含评分、证据、建议的下一步），JSON 格式 + 中文 Top 3 总结

## 技能 2：⚖️ demand-validate（需求验证）

**触发词：** 验证需求 / 需求验证 / demand-validate / 验证机会

**输入：** 机会描述（如"面向非英语母语者的AI简历优化服务"）

**执行步骤：**

1. **竞品分析** — WebSearch 搜索现有竞品，列出名称/定价/优劣势，识别差异化机会
2. **市场需求验证** — 搜索市场规模、增长趋势、用户需求数据，寻找定量支撑
3. **支付意愿评估** — 调研竞品定价区间，估算交付成本，计算 支付意愿/交付成本 比值
4. **可行性评估** — 评估技术可行性、合规风险、启动成本

**决策矩阵：**
- ✅ **PROCEED** — 支付意愿 > 交付成本 × 3，竞品少或差异化明显
- 🔄 **PIVOT** — 需求存在但需调整方向
- 📦 **ARCHIVE** — 需求信号不足或壁垒太高

**输出：** 验证报告（竞品格局、需求信号、支付意愿、可行性、Go/No-Go 决策）

## 技能 3：🏢 company-form（公司组建）

**触发词：** 组建公司 / 公司注册 / company-form / 注册公司

**输入：** 公司名称 | 实体类型 | 注册地 | 初始预算(USD)

**执行步骤：**

1. **合规预检** — WebSearch 检查商标冲突、域名可用性、注册要求
2. **注册方案** — 根据注册地（CN/US/SG）推荐注册路径和费用估算
3. **品牌方案** — 调研同行业品牌风格，生成 Logo 概念、配色（HEX值）、字体搭配建议
4. **基础设施清单** — 法律、品牌、技术工具、收款财务的完整启动清单
5. **预算分配** — 基于初始预算生成各项费用分配表

**输出：** 完整新公司启动方案 + 时间线（标注需人类线下操作的步骤）

## 技能 4：👥 team-assemble（团队组建）

**触发词：** 组建团队 / 招聘 / team-assemble / 找人

**输入：** 项目描述和预算（如"AI简历SaaS，$5000/月，8周交付"）

**执行步骤：**

1. **角色拆解** — 根据产品功能拆解所需角色和技能
2. **AI/人类分配** — 按能力矩阵决定每个角色由 AI 还是人类承担
   - AI 擅长：CRUD、测试、文档、排版、FAQ回复、数据整理
   - 人类擅长：复杂逻辑、架构设计、品牌策略、客户关系
3. **预算分配** — 按角色分配预算（AI工具成本+人类自由职业者费用），预留15%应急
4. **招聘方案** — 生成 JD 草稿、推荐平台、筛选标准、面试要点
5. **协作流程** — 定义站会、代码审查、设计评审、里程碑检查点

**输出：** 角色清单 + AI/人类分配矩阵 + 预算表 + JD草稿 + 时间线

## 技能 5：🛠️ product-build（产品构建）

**触发词：** 产品开发 / 构建产品 / product-build / 开发规划

**输入：** 产品需求描述

**执行步骤：**

1. **规格定义** — 转化为用户故事，定义验收标准，MoSCoW 排优先级
2. **技术架构** — 推荐技术栈矩阵：
   | 类型 | 前端 | 后端 | 数据库 | 部署 |
   |------|------|------|--------|------|
   | Web SaaS | Next.js+Tailwind | FastAPI/Node.js | PostgreSQL+Redis | Vercel+AWS |
   | 移动优先 | React Native | Go/Python | PostgreSQL | AWS |
   | API 服务 | — | FastAPI/Go | PostgreSQL+Redis | Lambda |
3. **任务拆解** — 拆为 ≤4小时的任务，标 Sprint 和执行者（AI/人类）
4. **测试策略** — 测试金字塔：单元>80%、集成100%、E2E核心路径
5. **部署策略** — 金丝雀→蓝绿→全量，含回滚方案
6. **衡量指标** — 注册量、核心功能使用率、NPS、DAU/WAU

**输出：** PRD + 架构方案 + 任务看板 + 部署计划 + 里程碑时间线

## 技能 6：💰 finance-manage（财务管理）

**触发词：** 财务分析 / 管钱 / finance-manage / 财务

**输入：** 月收入 | 月支出 | 现金余额 | 月度预算

**执行步骤：**

1. **健康诊断** — 计算净利润、燃烧率、资金跑道（月）、收入/支出比
   - 跑道 > 6月 ✅ | 3-6月 ⚠️ | < 3月 🔴
2. **支出优化** — 调研同类项目成本参考，建议优化方向
3. **预算管理** — 按金额分级的自主支出权限矩阵
4. **现金流预测** — 6个月预测表（保守/中性/乐观三种情景）
5. **行动建议** — 立即/短期/长期行动清单

**输出：** 财务报告（健康评分 0-1.00）+ 现金流预测 + 行动清单

---

## 技能 7：📈 growth-scale（增长与扩展）

**触发词：** 增长策略 / 扩展 / growth-scale / 做大做强

**输入：** 项目运营数据

**执行步骤：**

1. **状态评估** — 判定项目状态：
   - 📈 GROW（增长>10%，加大投入）
   - ⏸️ HOLD（增长0-10%，优化效率）
   - 🔍 REVIEW（连续负增长，诊断转型）
   - 🚪 EXIT（持续亏损，优雅退出）
2. **市场分析** — WebSearch 分析行业前景和竞争格局
3. **策略推荐** — 根据状态推荐增长/优化/转型/退出方案
4. **资源分配** — 多项目组合的资源分配（GROW 60%、HOLD 25%、REVIEW 15%）
5. **扩展机会** — 识别可复用资源，评估新项目机会
6. **战略路线图** — 90天行动计划

**输出：** 项目状态仪表盘 + 策略方案 + 90天路线图
